# Theory: Parallel BFS and DFS Traversal using OpenMP in C++

This practical implements Parallel Breadth-First Search (BFS) and Parallel Depth-First Search (DFS) traversal on a tree using OpenMP for parallel processing in C++. 

---

# Aim

To implement parallel BFS and DFS traversal of a tree using OpenMP in C++ and compare their execution times.

---

# Introduction

Tree traversal is a process of visiting all nodes of a tree data structure.

Two common traversal techniques are:

1. Breadth-First Search (BFS)
2. Depth-First Search (DFS)

In this practical, parallel processing is used to improve traversal performance using OpenMP multi-threading.

---

# What is Parallel Computing?

Parallel Computing is a technique where multiple tasks are executed simultaneously using multiple processors or threads.

Advantages:

* Faster execution
* Better CPU utilization
* Reduced processing time

OpenMP is used in this experiment for parallel execution.

---

# What is OpenMP?

OpenMP (Open Multi-Processing) is an API used for parallel programming in C/C++.

It supports:

* Multi-threading
* Parallel loops
* Shared memory programming

### Important OpenMP Directives Used

1. `#pragma omp parallel for`
2. `#pragma omp critical`

# Tree Data Structure

A tree is a hierarchical data structure consisting of:

* Nodes
* Edges

In this practical:

* Adjacency List representation is used.

Each node stores a list of connected nodes.

---

# Breadth-First Search (BFS)

## Definition

BFS visits nodes level by level.

It first visits:

* Root node
* All neighboring nodes
* Then next-level nodes

BFS uses a Queue data structure.

---

# Working of BFS

Steps:

1. Start from root node
2. Mark node as visited
3. Add node to queue
4. Remove node from queue
5. Visit all unvisited neighbors
6. Repeat until queue becomes empty

---

# Parallel BFS

In parallel BFS:

* Neighbor nodes are processed simultaneously using multiple threads.

### OpenMP Parallel Loop

```cpp
#pragma omp parallel for
```

Advantages:

* Faster traversal
* Reduced execution time

---

# Depth-First Search (DFS)

## Definition

DFS explores nodes deeply before backtracking.

It visits:

* One branch completely
* Then returns to explore another branch

DFS uses recursion or stack data structure.

---

# Working of DFS

Steps:

1. Visit root node
2. Mark node as visited
3. Visit one neighbor recursively
4. Continue until leaf node
5. Backtrack and explore remaining nodes

---

# Parallel DFS

In parallel DFS:

* Multiple neighboring branches are explored simultaneously.

Recursive calls are parallelized using OpenMP.

Advantages:

* Efficient branch exploration
* Faster execution for large trees

---

# Critical Section

When multiple threads access shared resources simultaneously, conflicts may occur.

To avoid this, critical sections are used.

### OpenMP Critical Directive

```cpp
#pragma omp critical
```

This ensures:

* Only one thread accesses shared data at a time.

Used for:

* Queue operations
* Updating visited nodes
