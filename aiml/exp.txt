# Experiment: HPC Application for AI/ML Domain using MPI K-Means Clustering

## Aim

To implement a High Performance Computing (HPC) application for AI/ML domain using MPI-based parallel K-Means clustering algorithm in C++.

---

# Theory

This experiment implements the K-Means clustering algorithm using MPI (Message Passing Interface) for parallel processing.

K-Means is a Machine Learning algorithm used for clustering similar data points into groups called clusters.

The program uses parallel computing concepts to improve execution speed by dividing the work among multiple processes.

This experiment combines:

* Artificial Intelligence / Machine Learning
* High Performance Computing (HPC)
* Parallel Processing
* MPI Programming

---

# What is K-Means Clustering?

K-Means is an Unsupervised Machine Learning algorithm.

It groups similar data points into K clusters based on distance from cluster centroids.

The algorithm works iteratively until the centroids stabilize.

---

# Why HPC is Used?

In large datasets:

* processing becomes slow on a single processor
* HPC improves speed using parallel execution

MPI divides the dataset among multiple processes so that computation happens simultaneously.

This reduces execution time.

---

# Working of Experiment

## Step 1: Initialize MPI Environment

MPI starts parallel execution using:

```cpp
MPI_Init()
```

Each process gets:

* rank (process ID)
* total number of processes

---

# Step 2: Generate Dataset

The root process (rank 0) creates random data points.

Each point contains:

* x coordinate
* y coordinate

Example:

```text
(12,45)
(67,89)
(25,31)
```

---

# Step 3: Initialize Centroids

Initial cluster centers (centroids) are selected from dataset points.

Example:

```text
Centroid 1 = First point
Centroid 2 = Second point
```

---

# Step 4: Broadcast Centroids

Centroid information is shared with all processes using:

```cpp
MPI_Bcast()
```

---

# Step 5: Scatter Dataset

Dataset is divided among processes using:

```cpp
MPI_Scatter()
```

Each process receives a subset of points.

---

# Step 6: Distance Calculation

Each process calculates distance between:

* local data points
* centroids

Distance Formula:

Nearest centroid determines the cluster assignment.

---

# Step 7: Update Centroids

Each process calculates:

* local sum of coordinates
* local count of points

Then all processes combine results using:

```cpp
MPI_Allreduce()
```

New centroid values are calculated.

---

# Step 8: Repeat Iterations

The process repeats multiple times until clusters become stable.

---

# Step 9: Final Output

Program prints:

* final centroids
* execution time

Example:

```text
Final Centroids:
Cluster 0: (25,40)
Cluster 1: (70,82)
```

---

# MPI Functions Used

## MPI_Init()

Starts MPI execution environment.

## MPI_Comm_rank()

Returns process ID.

## MPI_Comm_size()

Returns total number of processes.

## MPI_Bcast()

Broadcasts data to all processes.

## MPI_Scatter()

Distributes dataset among processes.

## MPI_Allreduce()

Combines results from all processes.

## MPI_Finalize()

Ends MPI execution.
